# Views — `src/views/`

> **Keep this file updated** whenever you add, remove, or significantly change a view.

Route-level pages built with Vue 3 Composition API + Ionic. Views compose stores, services, and reusable components into complete screens.

## View Inventory

| File | Purpose |
|---|---|
| `HomePage.vue` | Mixed home feed that combines communities, posts, and polls with new-content banners and runtime relay-aware loading. On mount it expands visible feed size up to an initial 50-item target, and it auto-expands again when newly loaded posts/polls arrive (up to that same cap), so fetched content is visible without first scrolling. It still uses incremental infinite-scroll loading beyond the cap. Includes detailed `[FeedDebug]` console diagnostics for warmup/subscription/render count tracing (enabled only when `localStorage.interpoll_feed_debug === 'true'`). To reduce startup Gun sync bursts, chat-room graph subscriptions are lazy-initialized when the user opens the Chat tab, and Home’s Gun feed subscriptions are opt-in (`localStorage.interpoll_home_gun_feed === 'true'`) with DB/bootstrap feed data as the default startup path. If that startup path returns an empty feed, Home now performs a short one-shot Gun recovery probe (first community only, timeout-capped) to repopulate without reintroducing broad startup fan-out. Desktop utility navigation includes direct access to the Resilience Center (no beta gate). The current UI refresh wraps the feed in the shared ambient page shell from `src/style.css`, with glass sidebars/cards and the sticky new-content banner layered inside the main scroll area. |
| `CommunityPage.vue` | Community detail page with feed, metadata, and join/share actions. In the mixed `all` filter, it now promotes one poll near the top when polls are present but would otherwise be buried below many posts, so polls are visible without deep scrolling. On direct route reloads it now performs a short `communityStore.loadCommunities()` bootstrap before selecting the community to reduce cold-start misses, and emits detailed `[CommunityDebug]` loading logs when `localStorage.interpoll_community_debug === 'true'`. |
| `PollDetailPage.vue` | Full poll page with inline vote submission, results gating, duplicate-vote checks, and receipt-first submission flow. Authorization now includes `poll.requireLogin`; if backend denies with auth-required, the page stores the current route and redirects to OAuth instead of marking the user as already voted. The backend authorization step returns a short-lived reservation token that the page passes to backend confirm after the chain vote (including login context), so only the same authorizing session can commit. After `chainStore.addVote()` succeeds it marks the vote locally, routes to the receipt immediately, and lets backend confirm plus Gun/MySQL follow-up sync continue in the background so slow or outdated relays do not bounce the user back into a second submission attempt. Invite and standalone vote links preserve `communityId` in the query string so community-scoped fallback loading still works on direct opens and refreshes. |
| `VotePage.vue` | Standalone vote route that loads a poll by id and renders `VoteForm.vue`; used for direct links and receipt-oriented vote flows. It now accepts optional `communityId` query context and passes it into the poll store so community-scoped polls still load when the global poll root is lagging. |
| `ReceiptPage.vue` | Displays a previously generated receipt verification code and linked vote details from the local chain. |
| `ResultsPage.vue` | Dedicated poll results page for viewing vote counts without the voting form. It now also honors optional `communityId` query context and preserves it when linking back into the standalone vote page. |
| `ChainExplorerPage.vue` | Local chain block explorer. Rendering is now progressive: it initially mounts a limited block window and loads more in batches, while signature verification is scoped to visible blocks so large sync bursts don’t force full-list DOM updates at once. |
| `SearchView.vue` | Search page for posts and polls; now relies on runtime relay config instead of capturing a fixed API base at mount time. |
| `ChatView.vue` | Route-reactive direct-chat page that reinitializes when the target peer changes without forcing a full page reload. |
| `JoinPrivatePage.vue` | Invite / private-community entry flow. Supports SPA navigation without a browser reload. |
| `ClaimUsernamePage.vue` | Username claim flow with optional trust-issuer verification. Loads issuer options from Gun + built-in/custom issuers, preselects the Endless issuer, supports adding a custom issuer endpoint from UI, and runs PoW-backed verified claim or unverified fallback. Verified claims now persist issuer-bound identity fields (`identityUsername`, `identityIssuer`, `identityTrustLevel`) so profile/post badges show trusted status consistently. |
| `SettingsPage.vue` | Runtime relay settings, diagnostics, and local app configuration. Network tab includes explicit bootstrap recovery (generate/copy/import bootstrap artifacts, Gun discovery seeding, probe-before-switch confirmation) plus known-server trust/freshness metadata (`local`/`peer`/`gun` source, signature status, TTL when provided). “Signed” reflects explicit verification state, not source labels. Generated bootstrap invites now also include optional browser handoff context (source peer id, connected server endpoint, and snapshot status counters such as posts/polls/block height), and import prompts surface that context while still requiring manual confirmation. Relay switching is blocked for offline probe results. “Currently connected” reflects the live open WebSocket URL, not config URL match alone. |
| `ResiliencePage.vue` | Dedicated resilience operations center for anti-censorship and continuity workflows. Provides quick actions (full relay scan, best-relay auto-switch, snapshot backup/share, copy relay report), detailed relay probing with censorship signals, relay add/switch/remove controls, snapshot import/export + peer sharing, and practical Tor/self-hosting guides. |
| `CreatePollPage.vue` | Poll composer route. Post action now exposes an explicit submit-in-progress state (`Posting…`) and validation toast messaging for incomplete form input, so users no longer see a silent no-op when required fields are missing or while the create request is in-flight. It also emits opt-in submit diagnostics when `localStorage.interpoll_poll_debug` includes `ui` (or `all`), so console logs can trace payload prep, create completion/failure, and total submit duration. |
| `CreatePostPage.vue` | Post composer route. The current UI refresh keeps the existing submit/image behavior but moves the page into the shared ambient shell from `src/style.css`, with a hero section, glass form container, and nested media panel so create flows visually match the refreshed home feed/cards. |
| `ProfilePage.vue` | User profile editor for display name/username/avatar and posting anonymity toggle. Shows explicit identity trust status from persisted profile identity fields (`identityTrustLevel` + `identityIssuer`) so verified issuer claims remain visible as `username@trust_issuer` across sessions and after username edits. Trusted badge styling remains green. |

## Notes

- These are route components; internal links should stay inside the SPA router rather than using full-page reloads.
- Poll voting UX is designed to prioritize the decentralized chain write and receipt generation. Relay-side confirmation and denormalized poll counters are follow-up sync tasks, not the success condition for the vote itself.
- Route-level UI refreshes should prefer the shared shell classes in `src/style.css` instead of re-implementing page backgrounds or card chrome inside each view.
